# Интеграция с Pandoc на Backend

Backend использует внешнюю утилиту Pandoc для преобразования входных документов из формата Markdown в исходный код Typst. Это является ключевым шагом в пайплайне генерации документов, позволяя пользователям работать с привычным Markdown, сохраняя при этом возможности Typst для высококачественной типографики.

## Модуль `app/utils/pandoc_converter.py` ([`app/utils/pandoc_converter.py`](../../app/utils/pandoc_converter.py))

Этот модуль предоставляет основную функцию для взаимодействия с Pandoc CLI.

### `markdown_to_typst(markdown: bytes) -> str`

Асинхронная функция, которая выполняет преобразование Markdown в Typst.

**Ключевые особенности:**

*   **Запуск подпроцесса:** Функция запускает команду `pandoc` в отдельном подпроцессе (`subprocess.run`). Используются флаги `--from=markdown+raw_attribute` (для обработки Markdown с необработанными атрибутами) и `--to=typst` (для указания выходного формата Typst).
*   **Входные данные:** Markdown-содержимое передается в Pandoc через `stdin` как байты.
*   **Выходные данные:** Результат (Typst-код) считывается из `stdout` и возвращается как строка.
*   **Обработка ошибок и тайм-аутов:** Функция перехватывает ошибки выполнения Pandoc (ненулевой код возврата) и ошибки превышения времени ожидания (`subprocess.TimeoutExpired`), выбрасывая кастомное исключение `PandocError`.
*   **Асинхронность:** Вызов Pandoc выполняется в отдельном потоке (`loop.run_in_executor`), чтобы не блокировать основной поток выполнения FastAPI, что важно для производительности асинхронного веб-сервера.

## Поток конвертации

Когда Backend получает Markdown-файл для генерации PDF или экспорта `.typ`:

1.  **Вызов функции:** [`app/api/generate.py`](../../app/api/generate.py) вызывает `markdown_to_typst`.
2.  **Запуск Pandoc:** В новом потоке запускается процесс `pandoc`.
3.  **Конвертация:** Pandoc обрабатывает Markdown-вход и генерирует соответствующий Typst-код.
4.  **Возврат результата:** Typst-код возвращается в [`app/api/generate.py`](../../app/api/generate.py), где он будет объединен с Typst-преамбулой и, возможно, с содержимым титульной страницы, для дальнейшей компиляции в PDF или экспорта.
