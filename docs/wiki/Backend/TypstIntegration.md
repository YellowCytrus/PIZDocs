# Интеграция с Typst на Backend

Сопутствующие **константы** (алиасы шрифтов, плейсхолдеры подписи, outdent нумерации) вынесены в [`app/utils/constants/typst_preamble_constants.py`](../../app/utils/constants/typst_preamble_constants.py) — обзор в [Constants.md](../Constants.md).

Backend активно использует Typst для генерации высококачественных документов в формате PDF. Интеграция включает в себя два основных аспекта: формирование Typst-преамбулы на основе стилей профиля и компиляцию полного Typst-кода в PDF с использованием CLI-утилиты Typst.

## Формирование Typst-преамбулы ([`app/utils/typst_preamble.py`](../../app/utils/typst_preamble.py))

Модуль `typst_preamble.py` отвечает за динамическое построение Typst-преамбулы. Преамбула — это набор глобальных настроек (`#set`) и правил отображения (`#show`) в Typst, которые определяют внешний вид документа (шрифты, отступы, нумерация, стили элементов и т.д.). Эти настройки формируются на основе стилевых профилей, хранящихся в базе данных.

### `build_typst_preamble(profile: Profile) -> str`

Эта функция является основной точкой входа. Она принимает объект `Profile` (профиль стилей) и, используя данные из связанных с ним стилевых моделей, генерирует строку, содержащую валидный Typst-код преамбулы. 

**Ключевые особенности:**

*   **Стилизация элементов:** Функция итерирует по различным стилевым объектам (`page_style`, `document_style`, `par_style`, `heading_style`, `bullet_list_style` и т.д.), связанным с профилем.
*   **`_typst_str` и `_typst_font`:** Вспомогательные функции для корректного форматирования строк и имен шрифтов для Typst (экранирование кавычек, обработка алиасов шрифтов).
*   **Обработка `HeadingLevelStyle`:** Для каждого уровня заголовка (H1-H6) генерируются специфичные `#show` правила, учитывающие такие параметры, как включение/выключение нумерации и принудительный разрыв страницы перед заголовком.
*   **Обработка `TextOverrideStyle`:** Поддерживается применение кастомных текстовых стилей к различным элементам с помощью `TextOverrideMixin` и `apply_text_override` ([`app/api/elements/text_override.py`](../../app/api/elements/text_override.py)), что позволяет переопределять шрифт, размер, цвет и другие текстовые свойства.
*   **Гибкость:** Модуль спроектирован так, чтобы быть легко расширяемым для добавления новых типов стилей или новых свойств к существующим стилям.

## Компиляция Typst в PDF ([`app/utils/typst_compiler.py`](../../app/utils/typst_compiler.py))

Модуль `typst_compiler.py` предоставляет функциональность для компиляции готового Typst-исходника в PDF-документ.

### `compile_typst_to_pdf(source: str, images_dir: Path | None, asset_files: dict[str, bytes] | None) -> bytes`

Эта асинхронная функция принимает полную строку Typst-кода и вызывает внешнюю утилиту `typst compile` через `subprocess` для выполнения компиляции.

**Ключевые особенности:**

*   **Временные файлы:** Для каждой компиляции создается временная директория, куда записывается Typst-исходник (`input.typ`) и, при необходимости, копируются изображения.
*   **Обработка изображений:**
    *   Функция `_extract_image_paths` извлекает локальные пути к изображениям из Typst-исходника.
    *   Локальные файлы изображений из `images_dir` (корневая директория проекта `/images/`) копируются во временную директорию перед компиляцией.
    *   Также поддерживается передача `asset_files` (словарь с относительными путями и бинарными данными), что используется для обработки изображений, загруженных через API `/upload-image` и хранящихся по ID.
*   **Выполнение `typst CLI`:** Команда `typst compile` запускается в подпроцессе. Обрабатываются возможные ошибки компиляции (возвращаются через `TypstCompileError`) и тайм-ауты.
*   **Асинхронность:** Компиляция выполняется в отдельном потоке (`loop.run_in_executor`), чтобы не блокировать основной поток FastAPI.
*   **Возврат:** Функция возвращает бинарные данные сгенерированного PDF-файла.
