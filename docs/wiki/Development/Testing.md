# Тестирование

## Backend: pytest

Тесты расположены в [`tests/`](../../../tests) и бьют по **публичному HTTP API** (без прямого импорта внутренних модулей, кроме вспомогательных фикстур).

**Требования:** доступная БД (часто `pizdo_test`); Postgres можно поднять так же, как в [настройке](Setup.md). Если БД недоступна, часть сценариев пропускается с пояснением.

### Команды

```bash
# Поднять только Postgres, если ещё не поднят
docker-compose up -d db

uv pip install -r requirements.txt
uv run pytest tests/
```

- Быстрые тесты **без** реального `pandoc`/`typst` (если отмечены `not integration`):
  ```bash
  uv run pytest tests/ -m "not integration"
  ```
- Интеграция с реальными `pandoc` + `typst` (маркер `integration`) — бинарники в PATH, иначе сценарий пропустится.
  ```bash
  uv run pytest tests/ -m integration
  ```

Точные команды и сброс тестовой БД см. в [README](../../../README.md) (раздел «Тесты») — дублировать сюда дословно не требуется, используйте README как оперативный чек-лист.

## Что проверяется (обзор)

Ниже — **карта покрытия** по смыслу; актуальное перечисление смотрите в `tests/`.

| Область | Примеры эндпоинтов / сценариев |
|--------|---------------------------------|
| Профили | `POST/GET/PATCH/DELETE /profiles` |
| Стили | чаще всего `bullet_list` как репрезентативный кейс; часть типов элементов может не иметь отдельного набора тестов |
| Титульные страницы | CRUD `/title-pages` |
| Преамбула | `GET /profiles/{id}/preamble` — 200, `text/plain`, 404 |
| PDF | `POST /profiles/{id}/generate-pdf` — заголовок PDF, 400/404/500, размер/расширение |
| Полный цикл | реальный md → PDF при наличии CI-бинарников |

**Известные пробелы (на момент выгрузки):** содержимое преамбулы детально не валидируется; не все типы `element` покрыты так же, как `bullet_list` — при добавлении фич дополняйте тесты. Расширенный чек-лист был в `docs/TEST_COVERAGE.md` (файл **заменён** переносом; историческая таблица интегрирована сюда).

## Frontend

В каталоге `frontend/`:

```bash
cd frontend && npm test
# или, в зависимости от настроек package.json, например:
npm run test:unit
```

Скрипты смотрите в [`frontend/package.json`](../../../frontend/package.json) (Vite + Vitest и т.д.).
