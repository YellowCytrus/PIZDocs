# Конвертация Markdown в Typst

Преобразование Markdown-текста в формат Typst является одним из первых и ключевых этапов в пайплайне обработки документов. Этот процесс может выполняться как на стороне Backend, так и на Frontend, в зависимости от потребностей (серверная генерация PDF или живой предпросмотр в браузере).

## Конвертация на Backend: Pandoc CLI ([`app/utils/pandoc_converter.py`](../../app/utils/pandoc_converter.py))

Backend использует внешнюю утилиту [Pandoc](https://pandoc.org/) для выполнения конвертации Markdown в Typst. Это обеспечивает высокую точность и поддержку широкого спектра возможностей Markdown.

### `markdown_to_typst(markdown: bytes) -> str`

*   **Механизм:** Функция запускает команду `pandoc` в отдельном подпроцессе.
*   **Параметры:** Используются флаги `--from=markdown+raw_attribute` (для правильной обработки Markdown с атрибутами) и `--to=typst`.
*   **Вход/Выход:** Markdown-текст передается через `stdin`, а Typst-код получается из `stdout`.
*   **Асинхронность:** Выполнение Pandoc происходит асинхронно, чтобы не блокировать основной поток FastAPI.
*   **Обработка ошибок:** Ошибки Pandoc (например, неверный формат) перехватываются и выбрасываются как `PandocError`.

### Когда используется

Этот метод используется на Backend для:

*   **Генерации PDF:** В эндпоинте `POST /profiles/{profile_id}/generate-pdf`, где сначала Markdown конвертируется в Typst, а затем полученный Typst-код компилируется в PDF.
*   **Экспорта `.typ`:** В эндпоинте `POST /profiles/{profile_id}/export-typ`, где возвращается чистый Typst-исходник.

## Конвертация на Frontend: `pandoc-wasm` ([`frontend/src/composables/usePandoc.ts`](../../frontend/src/composables/usePandoc.ts))

Для обеспечения живого предпросмотра в браузере, Frontend выполняет конвертацию Markdown в Typst непосредственно в браузере, используя WebAssembly-версию Pandoc — `pandoc-wasm`.

### `convertMdToTypst(markdown: string): Promise<string>`

*   **Механизм:** Функция использует библиотеку `pandoc-wasm` для вызова конвертации. Это WebAssembly-модуль, который позволяет выполнять логику Pandoc на стороне клиента, без запросов к серверу.
*   **Преимущества:** Значительно уменьшает задержку при предпросмотре, так как нет необходимости ждать ответа от Backend. Вся обработка происходит локально в браузере.
*   **Когда используется:** В компоненте [`Editor.vue`](../../frontend/src/views/Editor.vue) для обновления `typstSource` в `previewStore` каждый раз, когда пользователь вводит текст в Markdown-редактор.

## Сравнение подходов

| Аспект               | Backend (Pandoc CLI)                          | Frontend (`pandoc-wasm`)                                     |
| :------------------- | :-------------------------------------------- | :----------------------------------------------------------- |
| **Место выполнения** | Сервер (Python/FastAPI)                       | Браузер (JavaScript/WebAssembly)                             |
| **Инструмент**       | Внешняя утилита `pandoc`                      | Библиотека `pandoc-wasm`                                     |
| **Назначение**       | Генерация финальных PDF, экспорт `.typ`      | Живой предпросмотр в реальном времени                        |
| **Производительность** | Может иметь небольшую задержку из-за сетевых запросов и запуска подпроцесса | Высокая скорость, мгновенный отклик (после инициализации WASM) |
| **Изоляция**         | Отдельный процесс                             | Работает в песочнице браузера                                |

Оба подхода дополняют друг друга: Backend гарантирует стабильную и полную генерацию финальных документов, а Frontend обеспечивает интерактивный и быстрый пользовательский опыт при редактировании.
