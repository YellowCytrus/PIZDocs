# Обзор обработки документов

Данный раздел описывает сквозной процесс обработки документов в системе, начиная от ввода Markdown-текста пользователем и заканчивая генерацией PDF-файлов или живым предпросмотром Typst. Основной фокус — на взаимодействии различных компонентов и утилит для достижения конечного результата.

## Ключевые этапы обработки

1.  **Ввод и редактирование Markdown:** Пользователь создает и редактирует документ в Markdown-редакторе на Frontend. Этот Markdown-текст является исходным форматом.

2.  **Конвертация Markdown в Typst:** Markdown-текст преобразуется в промежуточный формат — Typst-код. Эта конвертация может происходить как на Frontend (для живого предпросмотра с помощью `pandoc-wasm`), так и на Backend (для серверной генерации PDF с помощью Pandoc CLI).

3.  **Формирование Typst-преамбулы:** На Backend генерируется Typst-преамбула — набор глобальных настроек и стилевых правил (`#set`, `#show`), основанных на выбранном профиле стилей. Эта преамбула определяет внешний вид всего документа.

4.  **Объединение и финальный Typst-исходник:** Typst-преамбула объединяется с сконвертированным Typst-кодом тела документа и, опционально, с кодом титульной страницы, чтобы сформировать полный итоговый Typst-исходник.

5.  **Рендеринг/Компиляция Typst:**
    *   **На Frontend (Live Preview):** Полный Typst-исходник передается WebAssembly-модулю `typst.ts`, который компилирует его и рендерит страницы на HTML Canvas для мгновенного предпросмотра в браузере.
    *   **На Backend (PDF Generation):** Полный Typst-исходник передается CLI-утилите Typst, которая компилирует его в высококачественный PDF-файл.

6.  **Управление изображениями:** Система поддерживает загрузку и вставку изображений в документы. Загруженные изображения хранятся на сервере, а ссылки на них включаются в Typst-код, будь то для предпросмотра или для генерации PDF.

## Сквозные потоки данных

*   **Живой предпросмотр в браузере:** Markdown (Frontend) → Typst Body (Frontend, `pandoc-wasm`) + Typst Preamble (Backend API) → Full Typst Source (Frontend) → Рендеринг (Frontend, `typst.ts` WASM) → HTML Canvas.
    *   Подробнее: [Пайплайн рендеринга Frontend](../Frontend/RenderingPipeline.md)
    *   Подробнее: [Поток преамбулы Typst](TypstPreambleFlow.md)

*   **Генерация PDF на сервере:** Markdown (Frontend) → Backend API → Typst Body (Backend, Pandoc CLI) + Typst Preamble (Backend) → Full Typst Source (Backend) → Компиляция (Backend, Typst CLI) → PDF (Backend) → Загрузка PDF (Frontend).
    *   Подробнее: [Потоки данных Backend](../Backend/DataFlows.md)
    *   Подробнее: [Typst в PDF](TypstToPDF.md)

## Взаимодействие с внешними утилитами

*   **Pandoc:** Используется для конвертации Markdown в Typst. Может быть вызван как на Frontend (через `pandoc-wasm`), так и на Backend (через Pandoc CLI).
    *   Подробнее: [Markdown в Typst](MarkdownToTypst.md)
*   **Typst CLI:** Используется на Backend для компиляции Typst-кода в PDF.
    *   Подробнее: [Typst в PDF](TypstToPDF.md)
*   **`typst.ts` (WASM):** Используется на Frontend для рендеринга Typst-кода в браузере.
    *   Подробнее: [Пайплайн рендеринга Frontend](../Frontend/RenderingPipeline.md)
